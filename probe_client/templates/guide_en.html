1. What is this probe?

This is a {{ brand }} probe — a local agent that monitors the network it is installed in and reports securely to the central {{ brand }} Command Center. Once paired it is driven remotely from the server, but you can also operate it from this local interface.

The top-bar LED shows the link to the server: green = online, grey/red = unreachable or not paired. Use the flag dropdown to switch language and the controls to change theme / font size.

2. Pairing (Dashboard)

1
On the Dashboard, fill in a probe name and, optionally, location and contact details.
2
Enter the Tenant name and the License Code provided by your administrator.
3
Click Connect to Server. An encrypted X25519/AES-GCM handshake runs and the probe becomes Paired & Secured.
4
From then on the probe auto-scans the network every 30 minutes and accepts commands from the server.

Factory Reset (Danger Zone) wipes all local keys and pairing — you will need a new License Code to reconnect.

3. Network Scanner

Under Network Scanner you can set the target subnet (CIDR, e.g. 192.168.1.0/24; leave blank to auto-detect), force an immediate scan, and watch the live debug logs. Discovered devices (IP, MAC, OS, open ports) are encrypted and sent to the server's Network Assets.

4. Wireless (WiFi / BLE)

The Wireless page scans nearby WiFi access points (SSID, channel, signal, security) and BLE devices (name, address). Results are shown locally and also forwarded to the server. WiFi needs a wireless interface; BLE needs a Bluetooth adapter accessible to the container.

5. Suricata IDS

1
Pick a capture interface (or let the server choose "auto").
2
Click Start. The probe downloads the active ruleset from the server and launches Suricata.
3
The Live Packet / Event Feed shows flows, DNS, TLS and alerts in real time; alerts are mirrored to the server's Threat Monitor.

Which rule categories are active is decided centrally on the server (IDS Rules page).

6. Remote control

Everything here can also be driven from the server's Probe Console: scans, Suricata start/stop, configuration, log fetch, packet capture, restart, self-update and factory reset. Sensitive commands are cryptographically signed.

7. Troubleshooting

  • LED red / "Server Unreachable" — check network connectivity to the server URL configured for the container.
  • Suricata won't start — ensure the package is installed and the container has NET_ADMIN/NET_RAW; check the feed for the error.
  • No BLE devices — the container needs Bluetooth access (privileged + host D-Bus, as in the bundled compose).
